## What to Know

The manuscript title is "RBC Component Factors and Transfusion-Associated Vital Parameter Changes in ICU Patients." Authors: Lucas D. Ekstrom and Gustaf Edgren, both affiliated with Karolinska Institutet, Stockholm. The study follows STROBE (Strengthening the Reporting of Observational Studies in Epidemiology) guidelines for cohort studies. The master manuscript file is `manuscript/MANUSCRIPT.md`, which is converted to DOCX via `convert_to_docx.py` for submission formatting. The manuscript follows IMRAD structure (Introduction, Methods, Results, Discussion) and maintains strict traceability between reported results, SAS analysis scripts, and dashboard visualizations. Each table and figure in the manuscript maps to a specific SAS script output. Clinical quantities are always reported with units and confidence intervals in parenthetical format. The manuscript presents both the primary transfusion effect findings and the null component factor results with equal rigor.

## Understanding

### Structure

The manuscript follows standard IMRAD organization:

- **Introduction**: Motivates the study by establishing what is known about RBC transfusion effects on hemodynamics in ICU patients, identifies the gap (limited data on donor/component characteristics modifying these effects), and states the study objectives. Approximately 3-4 paragraphs.
- **Methods**: Describes the SCANDAT3-S registry and Clinisoft data sources, the study population and inclusion criteria, variable definitions (vital parameters, component factors, covariates), and the statistical analysis approach (mixed-effects models, spline specifications, change estimation). This section maps primarily to the data preparation (Script 0) and model specification logic in Scripts 1-3.
- **Results**: Presents findings in the order: descriptive statistics (Table 1, from Script 1), primary transfusion-associated changes (Table 2 and Figure 1, from Script 2), and component factor interaction results (Table 3, from Script 3). Each result is anchored to the specific model (base or fully adjusted) that produced it.
- **Discussion**: Interprets the findings in context of prior literature, addresses the null component factor results directly, discusses strengths (large cohort, detailed time-series data, linked registry) and limitations (observational design, unmeasured confounders, single region), and concludes with implications for transfusion practice and future research.

The supplementary material includes additional tables for secondary vital parameters and sensitivity analyses.

### Table and Figure Correspondence

Traceability between the manuscript and the analytical pipeline is maintained as follows:

- **Table 1** (Baseline Characteristics): Generated by **Script 1** (`sas_code/01_descriptive_statistics.sas`). Presents patient demographics, ICU ward distribution, transfusion counts, and component factor distributions. Corresponds to the "Overview" tab in the dashboard.

- **Table 2** (Transfusion-Associated Vital Parameter Changes): Generated by **Script 2** (`sas_code/02_vital_trajectories.sas`). Two panels: (a) observed mean changes (unadjusted arithmetic means at pre/post time points) and (b) model-based estimated changes from the ESTIMATE statement in PROC MIXED (both base and fully adjusted). Corresponds to the "Vital Parameters" tab in the dashboard, which displays the trajectory curves.

- **Table 3** (Component Factor Effects on Vital Parameter Changes): Generated by **Script 3** (`sas_code/03_component_interactions.sas`). Reports the Type 3 interaction p-values for each factor x vital parameter combination, plus category-specific change estimates with CIs. Corresponds to the "Component Factors" tab in the dashboard.

- **Figure 1** (Vital Parameter Trajectories): Generated by **Script 2** output, visualized in the dashboard. Shows the spline-smoothed trajectory for each vital parameter over the 22-hour observation window, with the transfusion time marked at the origin. Typically displayed as separate panels for MAP, HR, SpO2, and other selected parameters.

Every number in the manuscript can be traced back to a specific SAS output dataset, which in turn maps to a specific script and model specification. This traceability chain is essential for reproducibility and revision accuracy.

### Style Conventions

The manuscript adheres to the following formatting and reporting standards:

- **Clinical quantities**: Always reported with units (e.g., "2.06 mmHg", "1.05 bpm", "0.3%"). No bare numbers for physiological measurements.
- **Confidence intervals**: Reported in parenthetical format: "2.06 (95% CI: 1.82, 2.30) mmHg". The CI is enclosed in parentheses with "95% CI:" prefix on first use, abbreviated to "CI:" thereafter.
- **P-values**: Reported to 2-3 significant figures. Values below 0.001 reported as "P < 0.001". Exact values preferred over threshold statements (e.g., "P = 0.03" rather than "P < 0.05") except for very small values.
- **STROBE adherence**: Each section of the manuscript maps to specific STROBE checklist items. The methods section explicitly describes the study design, setting, participants, variables, data sources, bias considerations, study size rationale, and statistical methods.
- **Null results**: Presented with the same precision and detail as positive findings. The discussion addresses potential reasons for null component factor findings without over-interpreting or dismissing them.
- **Abbreviations**: Defined at first use (e.g., "mean arterial pressure (MAP)"). Standard ICU abbreviations (HR, SpO2, FiO2) are defined once and used consistently.
- **No AI credit**: The manuscript does not include acknowledgment of AI assistance in drafting or analysis, consistent with current journal policies and author decisions.

### Conversion Pipeline

The manuscript source lives in `manuscript/MANUSCRIPT.md` as a Markdown file. Conversion to DOCX for journal submission uses the Python script `convert_to_docx.py`, which handles:

- Markdown-to-DOCX conversion with appropriate heading styles, table formatting, and reference placeholders.
- Formatting of tables to match journal requirements (borders, alignment, caption placement).
- Figure insertion with appropriate resolution and sizing.

The harmonization workflow that produces `MANUSCRIPT.md` follows these stages:
1. **Section drafts**: Individual sections (Introduction, Methods, Results, Discussion) are drafted and revised independently.
2. **Harmonization**: Sections are reviewed for consistency in terminology, abbreviation usage, tense, and voice. A harmonization pass ensures uniform style across sections.
3. **Transformation**: The harmonized text is formatted into the final Markdown structure with proper heading levels, table markup, and figure references.
4. **Merge**: All sections are combined into the single `MANUSCRIPT.md` file, which serves as the definitive source.
5. **Conversion**: `convert_to_docx.py` produces the DOCX output for submission.
